## Test Framework
Vitest (Vite native, Jest-compatible API). No config needed beyond `vite.config.ts`.
Install: `vitest`, `@testing-library/react`, `@testing-library/user-event`, `jsdom`

## Critical Unit Test Files

### `src/lib/parser.test.ts`
Highest priority — parser is the foundation everything else sits on.
- Parse CSV text → sorted Entry[] (verify ascending order by order column)
- Hand boundary detection: correctly groups entries into Hand objects
- Truncated last hand (no `-- ending hand --`): returns partial hand or skips
- All action regex patterns: fold, check, call, bet, raise, raise+allin, SB post, BB post, collected
- Board parsing: flop `[c1,c2,c3]`, turn `board [c4]`, river `board [c5]`
- Run-it-twice board detection: `Flop (second run)` lines captured as `board2`
- `Uncalled bet of N returned` correctly subtracts from net
- Hand result computation: won, lost, run-it-twice (sum both collected)

### `src/lib/parser.seat.test.ts`
Isolated seat position tests — this is the riskiest piece per the design.
- 2-player (BTN+BB, no SB): dealer = BTN, other = BB
- 3-player: dealer at seat #4, SB=#6, BB=#1 (wrap-around)
- 3-player: dealer at seat #6, SB=#1 (wrap), BB=#4
- 6-player: full rotation
- Player rebuy/leave mid-session: seat map updates correctly per-hand
- Seat #9 as dealer: SB wraps to lowest seat number

### `src/lib/stats.test.ts`
- VPIP: 5 hands, hero voluntarily enters 3 → 60%
- VPIP: BB check (no one raised) excluded from numerator
- PFR: hero raises 2 of 5 preflop → 40%
- AF: (2 bets + 1 raise) / 3 calls = 1.0
- WTSD: 4 hands saw flop, hero showed down in 2 → 50%
- Net: sum of hand results
- Empty hands array → all stats return 0, not NaN

### `src/lib/compress.test.ts`
- Round-trip: encode then decode returns identical hand object
- Unicode suit symbols survive compression (♥ ♦ ♣ ♠ must not corrupt)
- Corrupted URL string → returns null or throws cleanly, not silent wrong data
- Minimal hand (preflop fold only) → valid URL

### `src/lib/claude.test.ts`
Mock the Anthropic SDK for these tests.
- `chunkHands(120 hands, 50)` → returns 3 batches of [50, 50, 20]
- `buildPrompt(batch)` → output is valid JSON-parseable string, no curly braces in player names
- `parseClaudeResponse('[{...}]')` → correctly maps handId and tag
- `parseClaudeResponse('malformed json')` → returns [] not crash
- `parseClaudeResponse` with unknown tag value → filtered out
- `runParallelBatches`: all succeed → merged results
- `runParallelBatches`: one 429 → that batch retried, others not re-run
- `runParallelBatches`: one permanent failure → partial results returned

## Affected Pages/Routes

- `/` — Upload screen (drag-and-drop CSV, player picker)
- `/?session=<id>` — Session view (recent session loaded from IndexedDB)
- `/hand?data=<lzstring>` — Shared hand replayer (data from URL, no session needed)

## Key Interactions to Verify

- Upload a valid PokerNow CSV → player picker appears with all player names
- Select player from picker → session view loads with correct hand count
- Click any hand row → replayer opens showing correct board + seat positions
- Street navigation: Pre → Flop → Turn → River updates board cards and action log
- Share button on a hand → copies URL to clipboard (verify URL is valid)
- Open shared URL in fresh window (no CSV) → replayer loads correct hand
- AI scan completes → flagged hands appear as cards and highlighted rows
- Invalid API key → "API key error" banner, hand list remains browsable
- Rate limit hit mid-scan → partial results shown with "Scan incomplete" banner

## Edge Cases

- Upload a CSV for a run-it-twice hand → replayer shows both boards
- Upload CSV where hero is BTN (not SB/BB) in first hand
- Upload CSV with a player who left mid-session (seat disappears after hand N)
- Open shared URL with corrupted/truncated data → graceful error page, not blank
- Very large session (500+ hands) → IndexedDB save completes, AI scan batches correctly
- Session with zero VPIP hands → stats show 0%, not NaN or divide-by-zero

## Critical Paths

1. Upload CSV → pick player → session loads in <3 seconds
2. Share a hand → friend opens URL → sees full hand replay (no CSV needed)
3. AI scan with valid key → learning spots and hero calls flagged correctly
4. Session persists in IndexedDB → refresh page → "Recent Sessions" shows it
